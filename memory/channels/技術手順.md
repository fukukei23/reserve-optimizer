### ⚠️ コンテナの制約（2026-03-15確認）

- **コンテナ内からDocker操作はできない**
  - dockerコマンドが入っていない
  - Docker socketはあるが権限なし（nodeユーザー uid=1000）
  - sudoも入っていない
- **Dockerビルド・コンテナ再作成はVPS上で直接実行する**
- この手順はふくけいさんがVPSにSSH接続して実行する必要がある

### ブラウザ自動化の解決（2026-03-15）

**原因**: コンテナ内ではChromiumのサンドボックスが使えない
**解決策**: `openclaw.json`に`"noSandbox": true`を追加

```json
"browser": {
  "executablePath": "/usr/bin/chromium",
  "headless": true,
  "noSandbox": true
}
```

**結果**: browser toolが正常に起動するように（running: true, cdpReady: true）

### 自動マネタイズの方針（2026-03-15）

**推奨**: API優先で設計し、必要になったらブラウザに切り替え

理由：
1. APIは安定（UI変更に影響されない）
2. 高速・安価
3. 再利用性が高い
4. 二度手間にならない（APIロジックは流用可能）

**流れ**:
1. まずAPIで実装
2. API制限にぶつかったらブラウザに切り替え
3. 2FAが必要ならSurface Go起動

**Surface Go**: 「いつでも使える」状態にしておく（常時起動不要）

```bash
# 6. push確認
git push
```

### チャンネル間スキル認識問題（2026-03-15 19:50-20:15 UTC）

**症状**:
- #技術手順ではagency-agents自動発動
- #claw5速報では発動せず（mainのみ応答）

**調査結果**:
1. セッションは1つのみ（サーバー単位で管理）
2. `deliveryContext.lastTo`が#技術手順のまま更新されない
3. トランスクリプトに「aborted」（Request was aborted）が散見
4. 代理投稿は`allowBots: false`で処理されない

**原因候補**:
- gatewayの配信ハンドリング不整合
- セッション設計がサーバー単位（チャンネル差分なし）
- abortによる処理中断

**次のアクション**:
- ユーザーが#claw5速報で直接テスト投稿
- gatewayログ調査（要ホスト権限）

**追加調査（20:20-20:45 UTC）**:
- `allowBots: true` に設定変更（ボット自身のメッセージに反応する設定）
- 代理投稿は成功（messageId確認済み）
- **しかしボット自身のメッセージには反応しない**（Discordボットの仕様）
- ユーザー直接投稿であれば応答あり
- ツール呼び出し（web_search）は正常動作を確認

**結論**:
- `allowBots: true` は他のボットには有効だが、**自分自身のメッセージには無効**
- ユーザーが直接投稿すればスキル発動の可能性あり

**ステータス**: ユーザー直接投稿テスト待ち
