## 概要
このチャンネルは「OpenClawの運用方針（ops: operations / 運用）」をまとめ、チャネルの作り方・コンテキスト管理・自動化ルールを一元化するための場所です。フクロウ（bot）とふくけい（あなた）の二名のみが投稿・管理します。

## 最初に掲げるルール（案）
1. チャンネルの目的を明確にする
   - 新規チャンネルは「短い目的文（1行）」を作ってから作成する。
2. コンテキスト肥大化時の対処
   - 定期的（必要時）に自動要約して `memory/channels/<channel>.md` に保存する。
   - 長期履歴は要約を残し、元のメッセージはアーカイブ（そのまま残す）する。要約はLLMが重要情報を取り出しやすいフォーマットにする。
3. 新チャネルの作成基準（例）
   - 会話が現在トピックから外れて3往復以上続いたら分離を検討。
   - 永続的な作業単位（週次タスク、運用ルール、長期企画）は専用チャネルに移す。
4. 発動／スキル表記ルール
   - スキルAgentが応答する際は最初にスキル名＋emojiを名乗る（例: `# 👁️ Code Reviewer`）。
5. 投稿権限
   - 現在は「ふくけい（あなた）とフクロウ（bot）」のみ投稿可。重要投稿はbotが固定する。
6. 要約・アーカイブのフロー（自動化）
   - 定期または要求時にフクロウがチャネル全体を要約 → `memory/channels/<channel>.md` に保存 → 要約を新チャネルに投稿して固定。
7. 静穏時間（Quiet Hours）
   - 00:00–06:00 JST は自動通知を抑制（緊急のみ通知）。

## 運用テンプレ（投稿時の最低限）
- 投稿のタイトル（1行）
- 要点（3行まで）
- 関連ファイル/リンク（memory のパス）

## 過去会話の扱い（この作業の方針）
- 今回の移行は「自動要約 → memory/channels/openclaw-ops.md に保存 → あなたにレビュー提示」してから新チャネルへ投稿・固定します。
- 元ログはそのまま残します（復元可能）。必要なら別バックアップも作成します。
