# Obsidian Vault: フクロウ用ガイド

## ボルトの目的
- ふくけいのプロジェクトや日次メモを体系的に整理し、フクロウが素早く参照できるようにする。
- 情報はすべて日本語で記述する。

## フォルダ構成
- `00_Inbox` … 未整理メモ
- `01_Daily` … 日付ごとのログ（`YYYY-MM-DD.md`）
- `02_Projects` … プロジェクトごとの概要・タスク
- `03_Reference` … 汎用の資料、設定メモ
- `99_Archive` … 完了済み・保留中のもの

## 記述ルール
1. 各ノートの冒頭に目的や概要を記載する。
2. 重要語句や関連ノートには `[[ノート名]]` 形式でリンクを貼る。
3. タグは `#タグ名` を利用し、用途をそろえる（例: `#todo`, `#idea`, `#setup`）。
4. ふくけいからの指示や依頼は引用ブロック `>` で残し、経緯を追いやすくする。

## フクロウにやってほしいこと
1. ボルト全体を定期的にスキャンし、新規ノートや更新点を把握する。
2. プロジェクト単位で状況整理・要約を作成し、必要に応じて `02_Projects` に追記する。
3. 日次メモを読んだら重要ポイントを `memory/YYYY-MM-DD.md` に反映し、長期的に残すべき内容は `MEMORY.md` へ転記する。
4. リンク切れやタグの不整合を見つけたら修正提案を行う。

## 更新フロー
- 新しいメモはまず `00_Inbox` に置き、後で適切なフォルダへ整理する。
- プロジェクト完了後は `99_Archive` へ移し、`02_Projects` からのリンクを更新する。
- ボルト外でも重要な設定値やログは要約を `docs/` や `MEMORY.md` に残し、再利用しやすくする。
