## 基本方針

Claude Code の強みである「対話しながら形にしていく」を活かしつつ、最低限の品質を担保することを目指します。

### 品質管理の4本柱

1. **重要な判断の記録** - `docs/decisions/` に重要な決定を記録
2. **Gitコミットメッセージの充実** - What + Why の明確化
3. **定期的なコードレビュー** - 機能実装完了時または週次
4. **重要タスクの事前仕様** - Tier 1 タスクは簡易仕様を作成

## 並列エージェント機能

デスクトップ版 Claude Code では、`.claude/agents/` 配下のエージェントが自動読み込みされ、並列実行されます。

### 主な特徴
- **並列実行**: 複数のエージェントを同時実行可能
- **視覚的監視**: サイドバーで各エージェントの進捗を監視
- **タブ管理**: 各エージェントが別タブで管理される
- **バックグラウンド実行**: 長時間タスク中も作業継続可能

### 使い方
1. プロジェクトを Claude Desktop で開く
2. エージェントが自動読み込みされる
3. 自然言語で依頼すると、適切なエージェントが起動

## 使い方

### 1. 始め方
`/getting-started` スキルを実行して、基本的な使い方を確認

### 2. Tier 分類
`.claude/rules.md` に基づき、タスクが Tier 1（事前仕様必須）か判定されます

### 3. 開発フロー

#### Tier 2（通常フロー）
直接実装 → 自己レビュー → テスト計画

#### Tier 1（事前仕様必須）
簡易仕様確認 → 実装計画 → 安全チェック → 実装 → 自己レビュー → テスト計画 → コードレビュー
