# クリニック導入ガイド（reserve-optimizer）

> **目的**: reserve-optimizer をクリニックに導入する際の、**開発者（fukukei23）の作業**と**クリニック側の作業**を明確に分ける。
> **想定読者**: クリニック経営者・導入担当者・私自身（構築時のチェックリスト）

---

## 役割分担の全体像

| 作業領域 | 開発者（私） | クリニック側 |
|---------|------------|------------|
| LINE/Stripe/GAS の申請・認証 | ✅ 代行 | 必要情報の提供 |
| システムの構築・セットアップ | ✅ 全部 | - |
| 初期カスタマイズ | ✅ 代行 | ヒアリング回答 |
| スタッフ教育 | ✅ 初回のみ | 出席 |
| 日常運用（スプレッドシート更新等） | - | ✅ 全部 |
| 障害対応・調査 | ✅ | 連絡・情報提供 |
| 機能追加・改善 | ✅ 見積もり後 | 依頼・承認 |
| 顧客サポート | - | ✅ 一義対応 |

**核心**:
- 構築・技術対応はすべて私が代行
- クリニック側は「情報授受」「日常運用」「顧客サポート」を担当
- クリニック側に**コードを書く作業・複雑な設定作業は発生しない**

### ❌ クリニック側に発生しない作業

以下の作業は私（開発者）が全部代行するため、クリニック側では発生しません：

- ❌ コードを書く作業
- ❌ サーバ設定・API キーの取得
- ❌ スプレッドシートの初期構築
- ❌ LINE Messaging API の技術設定
- ❌ Stripe の技術設定
- ❌ Cloudflare Worker のデプロイ
- ❌ 障害の調査・修正作業

**クリニック側の作業は「日常のスプレッドシート更新」「顧客サポート」「緊急時の連絡」が中心**で、技術的な作業は発生しません。

## よくある質問（Q&A）

### Q. ランニングコストは？

| 項目 | 費用 | 発生条件 |
|-----|------|---------|
| GAS / Sheets / LINE メッセージ送受信 | **0円** | 標準的な予約数（月数百件）の場合 |
| Stripe 決済手数料 | 売上の 3.6% 程度 | 決済1件あたり |
| LINE 公式アカウント | 0〜数千円 | フリープラン200通/月超過時 |
| Google Workspace | 0円 | 一般アカウント利用時 |

**基本は月額 0円**で運用可能。決済手数料と LINE メッセージ超過時のみ課金。

### Q. 障害時はどうなりますか？

- 私の対応時間: 平日 9:00-18:00（時間外は翌営業日対応）
- 緊急度による:
  - **予約不能（全停止）**: 平日日中で 2時間以内対応
  - **一部機能停止**: 翌営業日対応
  - **軽微な表示崩れ**: 月次メンテナンスで対応
- 24時間緊急対応オプション: 別途見積もり

### Q. カスタマイズの費用は？

- **初期構築費**: 構築内容により要見積もり（基本パッケージ + 追加オプション）
- **運用開始後の追加カスタマイズ**: 工数ベースで別途見積もり（時給 or 案件単価）
- **小さな文言修正・営業時間変更**: 月次保守契約に含まれる範囲（無料）

### Q. 他のLINE予約システムへの移行はできますか？

- データは Google Sheets に保存されているため、エクスポートは容易
- 新システムへのデータ移行は 2-4 週間かかる想定
- スプレッドシート形式の CSV エクスポートは標準対応

### Q. セキュリティは大丈夫ですか？

- LINE / Stripe の Webhook で **HMAC-SHA256 署名検証** を実施（なりすまし防止）
- API キーは **GAS Script Properties** で暗号化保存（平文なし）
- Cloudflare Worker 経由でLINE/Stripe 双方を中継（二重防御）
- 月次セキュリティアップデートで最新仕様に対応

### Q. スタッフに LINE 操作が苦手でも大丈夫ですか？

- スタッフ向け説明会を実施（30分〜1時間）
- 主要操作は **3つのみ**（予約確認・キャンセル・変更）
- スプレッドシートは関数で自動集計されるため、関数知識不要

### Q. 土日・祝日も営業している場合は？

- 初期構築時に **営業日カレンダーを設定**（私が行います）
- 祝日は個別指定 or 「年末年始等の特定期間のみ休」に設定可能

### Q. 複数店舗ある場合は？

- 1店舗 ＝ 1デプロイ（同じソースを別 LINE アカウント・別 Sheets で運用）
- 複数店舗横断ダッシュボードは標準では非対応（別途開発）
