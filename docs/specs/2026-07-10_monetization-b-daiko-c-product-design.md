## 0. 前提・拘束条件

- **最優先ゴール**: 有償1件目を最速で作る（90日以内）
- **営業制約**: 低接触設計必須（DM可・商談未知数・現職あり）→ インバウンド受注・テキスト完結・匿名ハンドル
- **戦略**: B（構築代行）主軸 + C（プロダクト販売）並走。D（代行者卸し）はPhase 2・E（事業売却）は出口オプション。A（SaaS直販）は淘汰（保守月額として実質温存）
- **リスク前提**: 公務員の副業許可要否・確定申告の最終確認はユーザー自身の責任範囲（計画は「制約なし」回答前提）

## §1. 商品定義 — 「LINE予約の仕組みまるごと構築」

売るのは「ツール」ではなく「導入の手間の代行 + 月額ツール費の削減」。reserve-optimizerは納品原価を下げる自社ツール。

| パッケージ | 内容 | 価格 |
|---|---|---|
| ライト | LINE公式アカウント初期設定 + リッチメニュー + 予約Bot（予約/変更/キャンセル/リマインダー）+ スプレッドシート管理 | 29,800円 |
| スタンダード | ライト + Stripeデポジット決済（無断キャンセル対策）+ 口コミ依頼自動化（★4-5のみGoogle誘導） | 79,800円 |
| プレミアム | スタンダード + 多言語6言語 / AIチャット / スタンプカード / サブスクから2つ | 148,000円 |
| 月額保守 | 動作監視・文言変更・LINE仕様変更対応（6ヶ月無料→有料・任意） | 月4,980円 |

**訴求の核**（2026-07-15 転換・買い切り型→月額保守持続モデル）:
- 月額SaaS（ポチコ/リピッテ）より機能豊富で、**構築費＋月額保守の持続モデル**。6ヶ月無料で実績を積み、以降4,980円/月で長期サポート
- 「買い切り型・永久0円」は長期保守コストを抱え込む無計画な前提と判断し廃止（売り文句と実際の保守モデルの矛盾を解消）
- Stripeデポジット決済（無断キャンセル対策）は競合SaaSにほぼ無い
- Lステップ構築代行相場（30万円〜）より大幅に安い理由 = 自社ツール

## §2. 販売チャネル — 「営業しない営業」設計

1. **ココナラ出品（主戦場）**: 匿名ハンドル・インバウンド・チャット完結。初案件はレビュー獲得優先で29,800円。手数料22%は授業料と割り切る
2. **公開デモ（最重要営業資産）**: Phase α E11デモ環境を採用用・営業用で兼用。「触ってみてください」リンク1本が商談を代替
3. **X DM（サブ）**: 2026-04作成済みのDMテンプレ・検索クエリ再利用（SSOT `2026-04-19_ピボット-DMアウトリーチ準備.md`）。週10通・セラピスト/整体向け。決済・契約はココナラに誘導（個人間トラブル回避）
4. **note記事**: 月2本（例:「LINE予約を月額0円で作った話」）→ ココナラ出品への導線

## §3. C並走 — 納品副産物の資産化

- 納品時に必ず作るマニュアル・スクショ・手順を汎用化 → 「セルフ構築キット」（キット+動画+チャットサポート1週間）19,800円をBOOTH/ココナラ出品
- note有料記事（980円）でノウハウ切り売り → キット導線
- 追加工数は納品物の匿名化・汎用化のみ（1件あたり2〜3時間）

## §4. 数値計画

| 時期 | 目標 | 内訳 |
|---|---|---|
| 90日 | 有償1件目 | ココナラ初受注 29,800円 |
| 6ヶ月 | 月3〜5万円 | 構築1件/月 + 保守3件（約1.5万）+ note/キット |
| 12ヶ月 | 月10万円 | 構築1〜2件/月 + 保守8件（約4万）+ キット月1〜2本 |
| Phase 2（12ヶ月〜） | 月15万円〜 | D案発動: 代行者・Web制作者卸し（ライセンス5万円 or レベシェア）。LステップAPI連携アドオンも材料 |

保守月額のストック化により24ヶ月目以降「構築を止めても月5万」を目指す（実質SaaS化）。

## §5. 製品整備タスク（writing-plansで詳細化する対象）

納品1件の所要時間 = 利益率。ただし**出品が先・完璧な整備は後**（§6の実行リスク対策）。

1. **セットアップ自動化**: GAS複製→ScriptProperties設定→LINE/Stripe接続の半自動化スクリプト（目標: 納品作業3時間以内）
2. **業種汎用化**: 「初診/再診」→「初回/リピート」化・メニュー/営業時間の設定ファイル化（2026-04ピボット未解決課題の回収）
3. **公開デモ実装**: Phase α E11 spec流用（毎時リセット・GLM回数制限は設計済み）
4. **営業ページ**: GitHub Pages 1枚LP（デモリンク・料金・実績）
5. **法務**: docs/legal/（Plan A1のTERMS/PRIVACY/DISCLAIMER流用）+ ココナラ規約整合確認
6. **請求・契約**: 初期はココナラ内決済に一本化（請求書発行・入金管理を持たない）

## §6. リスクと対策

| リスク | 対策 |
|---|---|
| 公務員の名義・申告 | ココナラ匿名運用。許可要否・確定申告はユーザー自身で最終確認 |
| LINE/GAS仕様変更（全納品先に波及） | 保守契約の存在意義として訴求。保守10件で新規停止しマルチテナント化を判断 |
| 納品トラブル | サービス範囲外（LINE公式アカウントBAN・Stripe審査落ち等）を出品ページに明記 |
| 「また実行されない」（2026-04 DM未実行の再演） | 最初の2週間は整備でなく「ココナラ出品ページ下書き」から着手。90日計画を週次タスク化 |
