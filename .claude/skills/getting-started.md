# 始め方ガイド

## 基本操作

### 1. プロジェクト構造の確認
- `DEVELOPMENT.md` を読んで全体像を理解
- `.claude/agents/` 配下のエージェントを確認
- `.claude/skills/` 配下のスキルを確認

### 2. 開発ガイドラインの理解
`DEVELOPMENT.md` で定義されている4つの品質管理:
1. 重要な判断の記録
2. Gitコミットメッセージの充実
3. 定期的なコードレビュー
4. 重要タスクの事前仕様

## エージェントの自動化

### 自動読み込み
- `.claude/agents/*.md` を配置するだけで自動読み込み
- ユーザー操作や設定不要

### 自動起動条件
- **Code Reviewer**: 「レビューして」「コード確認して」
- **Decision Recorder**: 「〜にした」「〜を採用する」などの判断発言
- **Tier 1 Validator**: Tier 1 タスクのキーワードを含む指示

### 手動起動
- 明示的な指示でエージェントを起動

## 実装の基本フロー

### Tier 2 タスク（通常フロー）
```
ユーザーの指示 → 直接実装 → 自己レビュー → テスト計画
```

### Tier 1 タスク（事前仕様必須）
```
ユーザーの指示
  ↓
Tier 1 Validator が判定
  ↓
簡易仕様の確認（spec スキル）
  ↓
実装計画作成（plan スキル）
  ↓
安全チェック（safety-check スキル）
  ↓
実装
  ↓
自己レビュー（self-review スキル）
  ↓
テスト計画（test-plan スキル）
  ↓
コードレビュー（Code Reviewer エージェント）
```

## 最初のタスク推奨

1. プロジェクト構造の理解
2. Tier 2 タスクから始める（UIやバグ修正）
3. Gitコミットメッセージの作成（Whyを含める）
4. コードレビューの体験（並列エージェントの進捗監視）
