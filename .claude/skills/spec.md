# 要件確認スキル

## 使用タイミング
- Tier 1 タスクの実装前
- ユーザーから「仕様を確認して」「仕様書を作って」と言われた時

## 確認内容
以下の3-5点を箇条書きで確認:
1. 何を作るか
2. 主要な制約
3. 影響範囲

## 出力形式
```
簡易仕様を作成しました：

## タイトル
[機能名]

## 何を作るか
- [1-2文で説明]

## 主要な制約
- [技術的な制約]
- [性能要件など]

## 影響範囲
- [変更対象のファイルや機能]
```

## テンプレート保存先
必要に応じて `docs/specs/` に保存
