# テスト計画作成スキル

## 使用タイミング
- 実装後のテスト計画立案時
- 「テスト計画を作って」「どうテストするか」と言われた時

## 計画内容
### 単体テスト
- 各関数/メソッドのテスト項目
- 境界値のテスト

### 結合テスト
- モジュール間の連携確認
- 外部APIとの連携確認

### E2Eテスト（必要に応じて）
- ユーザー操作フローのテスト
- 主要なシナリオのテスト

## 出力形式
```
テスト計画を作成しました：

## 単体テスト
- [関数A]: [テストケース]
- [関数B]: [テストケース]

## 結合テスト
- [シナリオA]: [確認事項]
- [シナリオB]: [確認事項]

## 手動テスト（必要な場合）
- [操作手順]: [期待結果]

## テストデータ
- [必要なテストデータの準備]
```
