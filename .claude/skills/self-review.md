# 自己レビュースキル

## 使用タイミング
- コード実装完了後
- 「レビューして」「確認して」「自己レビューして」と言われた時

## レビュー観点
### 正確性
- 仕様通りに実装されているか
- エッジケースの考慮はあるか

### 可読性
- 変数名・関数名は適切か
- 不要な複雑さはないか

### 保守性
- 重複コードはないか
- 適切な抽象化がされているか

## 出力形式
```
自己レビュー結果：

### 良い点
- [良い実装箇所]

### 改善可能な点
- [改善案]

### バグリスク
- [潜在バグ]: [修正案]

### まとめ
[実装の品質評価: 良好/普通/要改善]
```
